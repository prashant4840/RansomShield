# AI-Powered Early Warning System for Android Ransomware

Startup-grade hackathon project demonstrating behavior-based Android ransomware early detection with modern SOC-like UI.

## Project Structure
- `android-app/` - Kotlin + Jetpack Compose Android app (MVVM, monitoring pipeline, explainable alerts)
- `ml/` - Python training scripts for anomaly detection
- `docs/` - Architecture and pitch assets

## Key Features Implemented
- Real-time telemetry stream and anomaly scoring pipeline
- Android signal collector adapters (CPU/memory/usage/activity heuristics)
- Adaptive sampling (faster scans during high-risk windows)
- Explainable risk scoring (Low/Medium/High)
- Smart alert context and in-app AI assistant guidance
- One-tap preventive actions (simulated isolation/lock/rollback/safe mode)
- Threat simulation mode for judge demos
- Futuristic cyber dashboard with timeline + app risk ranking + gamified score
- Temporal TFLite inference adapter (windowed reconstruction error) with safe fallback scoring
- Encrypted local threat logs + anti-tamper checks (debug/root/emulator signals)
- Firebase Firestore sync hook for multi-device event aggregation

## Build Android App
1. Open `android-app/` in Android Studio Hedgehog+.
2. Sync Gradle.
3. Run `app` on Android 8.1+ device/emulator.

### Local CLI prerequisites
- Java 17+ (`JAVA_HOME` configured)
- Gradle wrapper files (generated by Android Studio sync or `gradle wrapper`)

## Build from CLI (Windows PowerShell)
```powershell
cd android-app
.\gradlew assembleDebug
```

Output APK:
- `android-app/app/build/outputs/apk/debug/app-debug.apk`

## ML Training
```powershell
cd ml
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python train_model.py
```

Copy exported model into app assets:
```powershell
Copy-Item "artifacts\ransom_model.tflite" "..\android-app\app\src\main\assets\ransom_model.tflite" -Force
```

Metrics output:
- `ml/artifacts/metrics_report.json` (precision, recall, F1, false-positive rate)
